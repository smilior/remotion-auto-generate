# 要件定義書: Remotion Auto Generate

## 1. プロジェクト概要

### 1.1 目的

AI駆動開発に関する技術Tipsを30秒のショート動画として自動生成し、X（Twitter）で配信することで、著者の電子書籍へのマーケティング導線を構築する。

### 1.2 ゴール

Claude Codeにテキストで指示するだけで、ナレーション付きのターミナル風ショート動画が自動生成される一気通貫のパイプラインを構築する。

### 1.3 対象ユーザー

- 開発者本人（個人利用）

### 1.4 配信先

- X（Twitter）動画投稿

---

## 2. 動画仕様

### 2.1 コンテンツ

| 項目 | 仕様 |
|------|------|
| ジャンル | AI駆動開発の技術Tips |
| 長さ | 約30秒 |
| スタイル | ターミナル風画面にコード/コマンドがタイプされていく演出 |
| ナレーション | TTS音声あり |
| 字幕 | SRT字幕あり |
| BGM | フリー素材を使用 |

### 2.2 動画フォーマット（X最適化）

| 項目 | 仕様 |
|------|------|
| 解像度 | 1080×1920px（9:16 縦型） |
| フォーマット | MP4（H.264） |
| フレームレート | 30fps |
| 最大ファイルサイズ | 512MB（X上限） |
| 最大長 | 2分20秒（X上限、本PJは30秒） |

### 2.3 動画構成（暫定）

```
[0-3秒]   タイトル表示（Tipsのテーマ）
[3-25秒]  ターミナル風画面でコード/コマンドのタイプ演出 + ナレーション + 字幕
[25-30秒] エンディング（書籍導線 ※今後検討）
```

## 4. 機能要件

### 4.1 パイプラインステップ詳細

#### ① ナレーション原稿作成

- **入力**: ユーザーからのテキスト指示（技術Tipsの内容）
- **処理**: Claude AIが30秒のナレーション原稿を生成
- **出力**: ナレーション原稿テキストファイル
- **制約**: 30秒で読める分量（約100-120文字程度、日本語）

#### ② 音声生成

- **入力**: ナレーション原稿
- **処理**: Gemini TTS 2.5 Flash APIで音声合成
- **出力**: 音声ファイル（WAVまたはMP3）
- **備考**: 日本語対応、自然な読み上げ

#### ③ SRT字幕生成

- **入力**: 生成された音声ファイル
- **処理**: ローカルWhisperで音声認識・タイムスタンプ付き字幕生成
- **出力**: SRTファイル

#### ④ 動画レンダリング

- **入力**: ナレーション原稿、音声ファイル、SRTファイル
- **処理**: Remotionでターミナル風動画をレンダリング
- **出力**: MP4（音声なし or 音声込み）
- **演出要素**:
  - ターミナル風UI（黒背景、モノスペースフォント）
  - コード/コマンドのタイプライター演出
  - 字幕のオーバーレイ表示
  - タイトル画面・エンディング画面

#### ⑤ BGMミックス

- **入力**: レンダリング済みMP4、BGM音声ファイル
- **処理**: ffmpegでナレーション音声とBGMをミックス
- **出力**: 最終MP4ファイル
- **備考**: BGMはナレーションより音量を下げてミックス

### 4.2 実行インターフェース

- Claude Codeのチャットで指示するとパイプライン全体が実行される
- 例: 「Claude CodeのショートカットキーについてのTips動画を作って」

## 7. 開発フェーズ

### Phase 1: スモールサンプル作成（MVP）

各ステップを個別に動作確認しながら、1本の動画を手動パイプラインで作成する。

1. Remotionプロジェクトのセットアップ
2. ターミナル風コンポーネントの作成
3. ナレーション原稿の手動作成・TTS生成
4. Whisperでの字幕生成
5. ffmpegでのBGMミックス
6. Xへの試験投稿

### Phase 2: パイプライン自動化

Claude Codeからの一括実行を実現する。

1. 各ステップをスクリプト化
2. Claude Codeのスキル/フック化
3. エンドツーエンドの自動実行テスト

### Phase 3: 拡張（将来）

- 動画テンプレートのバリエーション追加
- 他プラットフォーム対応
- Web UI化（Next.js）
